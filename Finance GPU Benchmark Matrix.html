Platform Verdicts
▌ NVIDIA — Hopper & Blackwell
H200 SXM5
141GB HBM3e · 4.8TB/s · 67 TFLOPS FP64 · NVLink 4
  • Proven regulatory baseline — validated in SR 11-7, DORA, FINMA environments
  • Optimal for existing FP64 quant pipelines — lowest integration risk
  • Most cost-effective for pure Monte Carlo / DGEMM batch workloads
Verdict: Right-size for institutions not yet scaling to 70B+ LLM inference. Stable, auditable platform for quant and VaR.
B100 SXM
192GB HBM3e · 8TB/s · 90 TFLOPS FP64 · NVLink 5
  • 1.7× H200 memory bandwidth — strong uplift for XVA/CVA pipelines
  • NVLink 5 doubles peer bandwidth vs H200
  • Natural H100/H200 cluster refresh — no CUDA code changes
Verdict: Ideal evolutionary upgrade for banks on H100/H200. Best TCO for mixed quant + inference without full Blackwell capex.
B200 SXM
192GB HBM3e · 8TB/s · 112 TFLOPS FP64 · 2.25 PF BF16 · FP8
  • Highest single-GPU FP64 in standard Blackwell line — quant sweet spot
  • FP8 Transformer Engine enables 70B inference at production throughput
  • Structured sparsity for fraud/AML model acceleration
Verdict: The 2025-2026 single-GPU sweet spot — balances quant FP64 depth and AI inference breadth without NVL72 rack commitment.
B300 SXM
288GB HBM3e · 16TB/s · ~140 TFLOPS FP64 est. · ~2.8 PF BF16 est.
  • 288GB VRAM + 16TB/s — GB300-class specs in standard server form factor
  • Estimated best single-GPU for STREAM-bound risk analytics
  • ~2× B200 on memory bandwidth — XVA book fits in-GPU without spill
Verdict: Best single-server option for institutions needing GB300 memory without full NVL72 capex. Pre-GA — estimates only.
GB200 NVL72
192GB · 8TB/s/GPU · 2.5 PF BF16 (rack) · NVLink 5 · 130TB/s fabric
  • NVL72 rack as single GPU abstraction — 70B–405B models fully in-rack
  • Real-time intraday XVA/CVA recalculation becomes operationally viable
  • Transformative for multi-model trading signal ensembles
Verdict: Tier-1 bank inflection point. Justify capex by consolidating LLM inference, stress testing, and XVA onto one fabric.
GB300 NVL72
288GB · 16TB/s/GPU · ~4 PF BF16 (rack) est. · ~200TB/s fabric est.
  • 288GB × 72 GPUs = 20TB+ in-rack VRAM — all next-gen model sizes covered
  • 16TB/s/GPU eliminates every memory bottleneck in finance
  • FRTB IMA full-book intraday recalc becomes feasible for largest trading books
Verdict: Sovereign / tier-1 future platform. Justify with multi-workload consolidation. Pre-GA — estimates only.
▌ AMD — Instinct MI Series
MI300X
192GB HBM3 · 5.3TB/s · 84 TFLOPS FP64 · 1.3 PF BF16 · Infinity Fabric v3
  • Largest VRAM at launch — drove LLM inference adoption before Blackwell
  • Competitive FP64 vs H200 — viable for quant HPC workloads
  • Open ROCm/HIP stack — attractive for banks with open-source compliance mandates
  • Deployed at scale by major hyperscalers — production-proven ecosystem
Verdict: Credible H200 alternative. Best AMD case is LLM inference (VRAM headroom) and open-stack SR 11-7 model validation.
MI350X
288GB HBM3e · ~8TB/s · ~108 TFLOPS FP64 est. · ~1.8 PF BF16 est.
  • 288GB matches B300/GB200 — AMD's direct Blackwell memory response
  • ~1.6× MI300X bandwidth uplift — strong for XVA and VaR batch pipelines
  • FP8 support expected — inference competitiveness improves significantly
Verdict: Compelling for multi-vendor diversification under DORA operational resilience requirements. Pre-GA.
MI355X
288GB HBM3e · ~8TB/s · ~116 TFLOPS FP64 est. · ~2.0 PF BF16 est.
  • AMD flagship — ~2 PF BF16 approaches B200 inference territory
  • Optimised inference efficiency at lower TDP vs equivalent NVIDIA SKUs
  • Infinity Fabric v4 improvements address multi-GPU scale-out gap
Verdict: AMD's top finance GPU. Best for vendor-diverse estates or non-NVIDIA inference at scale. Pre-GA estimates.
Tokenization & Throughput — Platform Verdicts
▌ NVIDIA — Tokenization Highlights
H200 SXM5
141GB · No FP8 · ~18K prefill tok/s · ~96K max ctx (70B)
  • Sufficient for batch overnight regulatory NLP — not for real-time inference at scale
  • No FP8 Transformer Engine — highest per-token compute cost in NVIDIA range
  • 96K context handles most individual contracts and regulatory circulars
Tokenization Verdict: Viable for non-latency-sensitive NLP. Upgrade path to Blackwell required for high-concurrency or long-context finance workloads.
B200 SXM
FP8 TE · ~45K prefill tok/s · ~1.9× FP16 throughput · 160K ctx
  • FP8 Transformer Engine: ~1.9× throughput uplift vs FP16 — best single-GPU TCO for AML/NLP
  • 160K context covers full ISDA master agreements and Basel IV chapters in one pass
  • ~115 tok/s single-user decode — real-time analyst-facing applications viable
Tokenization Verdict: 2025-2026 sweet spot for financial NLP. FP8 efficiency + context headroom suits compliance automation and regulatory RAG pipelines.
GB200 NVL72
NVL72 fabric · 1M+ token ctx · 360+ concurrent slots · ~82K prefill tok/s
  • 1M+ token context enables full-book regulatory corpus analysis as a single prompt
  • 360+ concurrent 128K sessions — entire trading floor compliance queries in parallel
  • NVL72 unified memory eliminates KV cache spill — latency fully deterministic
Tokenization Verdict: Transformative for tier-1 institutions. Consolidate all LLM tokenization workloads — inference, RAG, fine-tuning — onto single NVL72 fabric.
▌ AMD — Tokenization Highlights
MI300X
192GB HBM3 · Partial FP8 · ~22K prefill tok/s · 160K ctx
  • 192GB VRAM: large KV cache advantage — first GPU to run 70B fully in-GPU at launch
  • Competitive prefill vs H200 — strong for document-heavy batch NLP workloads
  • ROCm/vLLM stack matures rapidly — viable for open-source compliance toolchains
Tokenization Verdict: Credible H200 alternative for long-context batch NLP. FP8 gap limits peak throughput vs Blackwell — best in open-stack environments.
MI355X
288GB HBM3e · FP8 support · ~46K prefill tok/s est. · 280K ctx est.
  • 288GB VRAM — matches B300 context capacity, competitive 280K token window
  • FP8 throughput estimated at ~1.8×: approaches B200 efficiency for inference workloads
  • Lower TDP profile — favourable for energy-cost-sensitive European banking data centres
Tokenization Verdict: AMD's strongest tokenization platform. Pre-GA estimates — validate FP8 maturity in ROCm stack before committing to production NLP pipelines. Est. only.
Tokenization data notes: Prefill and decode throughput figures for Llama3-70B at FP16/BF16 (or FP8 where noted) derived from published vLLM benchmarks, TensorRT-LLM benchmarks, MLPerf Inference v4 server results, and NVIDIA/AMD product disclosures. KV cache slot estimates assume 128K context window with KV cache in BF16 at 70B parameter model (approx. 16GB per 128K slot at BF16). Max context estimates assume full KV in-VRAM without offloading. TTFT (time-to-first-token) estimates for 4K token prompt on a single GPU without tensor parallelism. ⚠ B300, GB300, MI350X, MI355X tokenization figures are architectural projections — not independently validated. FP8 throughput multipliers are indicative; real-world uplift depends on model architecture, quantisation quality, and serving framework maturity (TRT-LLM vs vLLM vs ROCm). AMD FP8 support on MI300X is partial via ROCm; MI350X/MI355X FP8 support assumed full but unconfirmed. NVL72 context estimates assume unified fabric KV cache distribution across all 72 GPUs.
Data sourcing & caveats (benchmark matrix): NVIDIA H200, B100, B200, GB200 figures from published NVIDIA whitepapers, MLCommons MLPerf v4 results, GTC 2024/2025 announcements, and NVIDIA product briefs. AMD MI300X from AMD product brief and published MLPerf submissions. ⚠ B300, GB300, MI350X, MI355X figures are architectural projections (marked est.) — not yet validated in independent third-party benchmarks. Relative multipliers (×H200) are indicative for workload planning and not a substitute for PoC testing. Note: GB200/GB300 NVL72 per-GPU FP64 figures match B100/B300 respectively — their advantage is NVL72 rack-scale fabric (130–200TB/s), not per-GPU FP64 uplift. AMD Infinity Fabric bandwidth does not match NVLink NVL72 at rack scale — a significant architectural distinction for large-model finance workloads.
